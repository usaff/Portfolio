Airline Flight Status Analysis Dashboard - README

Airline Flight Status Analysis Dashboard

This report provides a comprehensive analysis of airline delays and cancellations, helping stakeholders understand key trends, identify problem areas, and optimize operations. It helps airlines, airports, and policymakers make data-driven decisions to improve on-time performance and reduce cancellations.

Inspiration and Motivation

With the increasing dependence on air travel for personal and business needs, flight delays and cancellations can cause significant inconvenience and financial losses. This project was inspired by the desire to understand the patterns behind these disruptions and find ways to minimize them.

By analyzing historical flight data, we aim to provide airlines with the tools to enhance punctuality and improve customer experiences. This isn’t just about data analysis—it's about making air travel more reliable and stress-free for millions of passengers worldwide.

Purpose of the Project

The primary goal of this project is to create an interactive Power BI dashboard that provides a comprehensive overview of airline flight statuses, focusing on:

  • Identifying patterns in flight delays and cancellations.
  • Understanding the impact of different factors (e.g., City, Airline, Day of the Week) on flight performance.
  • Supporting airlines and stakeholders in making data-driven operational decisions.

Steps Taken

1. Data Preparation & Cleaning

✔ Imported data from various sources including Excel files and databases.
✔ Cleaned the data by removing duplicates, handling missing values, and standardizing date formats.
✔ Created relationships between datasets for accurate analysis.

2. Data Transformation

✔ Used Power Query to format flight details, categorize delay reasons, and extract insights.
✔ Transformed raw timestamps into Year-Month-Day formats for trend analysis.
✔ Mapped categorical fields like Airline Name, City, and Delay Reasons to improve readability.

3. Data Modeling

✔ Built a star schema with a fact table (Flight Delays) and dimension tables (Airlines, Cities, Dates).
✔ Established one-to-many relationships between the fact and dimension tables.
✔ Added calculated columns and measures using DAX.

4. Visualization & Dashboard Creation

✔ Main Dashboard: Airline Delays & Cancellations Insights
✔ KPIs: Total Flights, Delayed Flights, Canceled Flights
✔ Line Charts: Flight Trends, Monthly Delays & Cancellations
✔ Bar Charts: Delays by Airline, City-wise Flight Count
✔ Pie Chart: Cancellation Reasons Breakdown
✔ Matrix: % Canceled by Day of the Week
✔ Applied slicers for dynamic filtering by Airline, City, and Date.

Key DAX Formulas

Total Flights

    TotalFlights = COUNT(Flights[FlightID])

✔ Counts the total number of flights in the dataset.

Delayed Flights

    DelayedFlights = CALCULATE(COUNT(Flights[FlightID]), Flights[Status] = "Delayed")

✔ Filters only flights with a "Delayed" status and counts them.

% On-Time Flights

    OnTimePercentage = DIVIDE(
        CALCULATE(COUNT(Flights[FlightID]), Flights[Status] = "On-Time"), 
        [TotalFlights]
    )

✔ Calculates the percentage of flights that arrived on time.

Cancellations by Reason

    CancellationsByReason = CALCULATE(
        COUNT(Flights[FlightID]), 
        Flights[Status] = "Canceled", 
        Flights[Reason] = "Weather"
    )

✔ Counts flights canceled due to specific reasons like weather.

Flight Status Calculations

    On-Time Flights = CALCULATE([Total Flights], Flights[Status] = "On-Time")
    
    Cancelled Flights = CALCULATE([Total Flights], Flights[Status] = "Cancelled")
    
    Delayed Flights = CALCULATE([Total Flights], Flights[Status] = "Delayed")

Percentage Calculations for Flight Status

    % Cancelled = DIVIDE([Cancelled Flights], [Total Flights], "-")
    
    % Delayed = DIVIDE([Delayed Flights], [Total Flights], "-")
    
    % On-Time = DIVIDE([On-Time Flights], [Total Flights], "-")

✔ Provides percentage calculations for each flight status.

Performance Optimizations

  • Optimized Data Model: Reduced table size by removing unnecessary columns and normalizing relationships.
  • Indexing & Aggregations: Used summarized tables to speed up calculations for KPIs.
  • Efficient DAX Queries: Applied SUMX, DIVIDE, and CALCULATE efficiently to minimize performance bottlenecks.
