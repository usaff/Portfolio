Analyzing Car Sales Trends

Introduction (Why We Are Doing This)

Understanding car sales trends is essential for assessing market demand, pricing patterns, and fuel type preferences. Raw data alone does not provide actionable insights, so this project leverages SQL queries to analyze car availability, sales trends, and key factors affecting pricing. By structuring and processing the data, we can uncover meaningful patterns that help businesses and consumers make informed decisions.

Project Description

This project involves analyzing car sales data using SQL queries to explore key metrics such as car availability by year, fuel type distribution, and pricing trends. It includes queries to determine the total number of available cars, track the impact of fuel types over time, and identify high-performing car models. The analysis helps in understanding depreciation trends, brand performance, and customer preferences in the automotive market. 


CREATE SCHEMA cars;
USE cars;

SELECT *  
FROM `car dekho`;

SELECT count(*) FROM `car dekho`;

Explanation:

The COUNT(*) function counts the total number of rows in the car dekho table. It does not filter anything, meaning it gives the total number of cars listed in the database.

üí° Why do this? This helps determine the total number of records in the dataset. If the number is too large or too small, it could indicate data inconsistencies or missing records. 


-- HOW MANY CARS WILL BE AVAILABLE IN 2023 --
SELECT count(*)
FROM `car dekho`
WHERE `year` = 2023;

Explanation:

The WHERE year = 2023 condition filters the table to include only cars manufactured in 2023.

COUNT(*) then counts the number of rows that meet this condition.

üí° Why do this? This helps determine how many new cars from the latest model year are available. 


-- HOW MANY CARS ARE AVAILABLE IN 2020, 2021, 2022 --
SELECT count(*)
FROM `car dekho`
WHERE year IN (2020, 2021, 2022)
GROUP BY `year`;

Explanation:

The WHERE year IN (2020, 2021, 2022) condition filters cars from these specific years.

GROUP BY year groups the results by each year.

COUNT(*) then counts how many cars belong to each of these years separately.

üí° Why do this? This provides a breakdown of cars available for these years, helping compare trends. 


-- PRINT THE TOTAL OF ALL CARS BY YEAR --
SELECT `year`, count(*)        
FROM `car dekho`
GROUP BY `year`;

Explanation:

GROUP BY year groups the cars based on their manufacturing year.

COUNT(*) calculates the total number of cars for each year.

üí° Why do this? This allows us to analyze trends and see which years have more listed cars. 


-- HOW MANY DIESEL CARS WILL BE AVAILABLE IN 2020 --
SELECT count(*)
FROM `car dekho`
WHERE `fuel` = 'diesel' AND `year` = 2020;

Explanation:

WHERE fuel = 'diesel' filters the table to include only diesel cars.

AND year = 2020 ensures that only cars from 2020 are counted.

üí° Why do this? This helps analyze fuel trends and availability. 


-- PRINT ALL THE FUEL CARS (PETROL, DIESEL, CNG) BY YEAR --
SELECT `fuel`, `year`, COUNT(*)
FROM `car dekho`
GROUP BY `fuel`, `year`
ORDER BY `fuel`, `year`;

Explanation:

GROUP BY fuel, year groups cars based on both fuel type and year.

COUNT(*) counts the number of cars within each group.

üí° Why do this? This helps analyze fuel trends over the years. 


-- IDENTIFY THE YEAR WITH MORE THAN 100 CARS --
SELECT `year`, COUNT(*) AS total_cars
FROM `car dekho`
GROUP BY `year`
HAVING total_cars > 100;

Explanation:

HAVING total_cars > 100 filters years where more than 100 cars are listed.

üí° Why do this? This highlights high-production years. 


-- ALL CARS COUNT DETAILS BETWEEN 2015 AND 2023 --
SELECT `year`, COUNT(*) 
FROM `car dekho`
WHERE `year` BETWEEN 2015 AND 2023
GROUP BY `year`
ORDER BY `year`;

Explanation:

The BETWEEN 2015 AND 2023 condition filters data for cars manufactured between these years.

GROUP BY year ensures we count cars for each specific year separately.

ORDER BY year sorts the output in ascending order, making trends clearer.

üí° Why do this? This helps analyze the number of cars available per year and understand how production fluctuates over time. 


-- Which car brands have the highest average selling price? --
SELECT Name, AVG(selling_price) AS avg_selling_price
FROM `car dekho`
GROUP BY Name
ORDER BY avg_selling_price DESC;

Explanation:

AVG(selling_price) calculates the average selling price for each car brand.

GROUP BY Name groups results by car brand.

ORDER BY avg_selling_price DESC sorts the brands with the highest average prices at the top.

üí° Why do this? This identifies premium brands and helps understand pricing trends across different manufacturers. 


-- What is the average selling price by fuel type and transmission type? --
SELECT fuel, transmission, AVG(selling_price) AS avg_selling_price
FROM `car dekho`
GROUP BY fuel, transmission
ORDER BY fuel, transmission;

Explanation:

This query groups cars by both fuel type (Petrol, Diesel, CNG) and transmission type (Automatic, Manual).

AVG(selling_price) calculates the average selling price for each combination.

ORDER BY fuel, transmission organizes the output neatly for analysis.

üí° Why do this? This helps compare whether automatic or manual cars tend to have higher prices, and how fuel type affects value. 


-- Cars manufactured in 2020 or later with km_driven < 20,000, sorted by selling price descending --
SELECT `Name`, selling_price, km_driven, `year`
FROM `car dekho`
WHERE `year` >= 2020 AND km_driven < 20000
ORDER BY selling_price DESC;

Explanation:

The WHERE year >= 2020 filter ensures we only select newer cars.

km_driven < 20000 ensures low-mileage cars are selected.

ORDER BY selling_price DESC sorts the most expensive cars at the top.

üí° Why do this? This helps find high-value used cars with minimal usage. 


-- Identify the top 5 cars with the highest max_power (bhp) and their corresponding mileage --
SELECT `Name`, max_power, mileage 
FROM `car dekho`
ORDER BY max_power DESC
LIMIT 5;

Explanation:

ORDER BY max_power DESC ranks cars by their horsepower.

LIMIT 5 ensures only the top 5 results are displayed.

üí° Why do this? This helps identify the most powerful cars in the dataset. 


-- Compare the average km_driven and selling_price for cars sold by "Dealer" vs "Individual" --
SELECT seller_type, AVG(km_driven) AS avg_km_driven, AVG(selling_price) AS avg_selling_price
FROM `car dekho`
GROUP BY seller_type;

Explanation:

AVG(km_driven) calculates the average distance driven for each seller type.

AVG(selling_price) determines how prices vary between dealers and individuals.

GROUP BY seller_type categorizes data by seller type.

üí° Why do this? This helps determine whether dealers or individual sellers offer better value. 


-- Categorize cars as "Premium", "Family", or "Budget" based on engine size, seats, and fuel type --
CREATE VIEW car_category AS
SELECT Name,
`engine`,
`seats`,
`fuel`,
CASE
WHEN `engine` > 2.5 AND seats > 5 AND `fuel` = 'Petrol' THEN 'Premium'
WHEN `engine` BETWEEN 1.5 AND 2.5 AND seats BETWEEN 4 AND 5 AND `fuel` IN ('Diesel', 'Petrol') THEN 'Family'
ELSE 'Budget'
END AS category
FROM `car dekho`;

Explanation:

The CASE statement categorizes cars based on:

  • Premium: Large engine (>2.5L), more than 5 seats, Petrol.
  • Family: Mid-range engine (1.5L-2.5L), 4-5 seats, Diesel or Petrol.
  • Budget: All other cars.

CREATE VIEW stores this categorization as a virtual table for future use.

üí° Why do this? This helps classify cars into meaningful segments for buyers. 


-- Year-over-Year Trend Analysis (Selling Price & Mileage) --
SELECT `year`, AVG(selling_price) AS avg_selling_price, AVG(mileage) AS avg_mileage
FROM `car dekho`
GROUP BY `year`
ORDER BY `year` DESC;

Explanation:

AVG(selling_price) calculates the average price of cars for each year.

AVG(mileage) gives the average mileage (fuel efficiency) of cars per year.

ORDER BY year DESC ensures the most recent data appears first.

üí° Why do this? This helps analyze pricing trends and improvements in fuel efficiency over time. 


-- Top 5 Cars with Best Mileage and Power --
SELECT `NAME`, mileage , max_power
FROM `car dekho`
ORDER BY mileage DESC, max_power DESC
LIMIT 5;

Explanation:

ORDER BY mileage DESC, max_power DESC ranks the most fuel-efficient and powerful cars at the top.

LIMIT 5 ensures only the top 5 results are displayed.

üí° Why do this? This helps buyers find cars with the best fuel efficiency and performance. 


-- Calculate depreciation rate of cars (2025) by fuel type --
SELECT fuel, `year`,
AVG(selling_price) AS avg_selling_price,
(MAX(selling_price) - MIN(selling_price)) / (2025 - MIN(year)) AS avg_depreciation_rate
FROM `car dekho`
GROUP BY `year`, fuel;

Explanation:

This query calculates the depreciation rate for cars by fuel type.

The formula (MAX(selling_price) - MIN(selling_price)) / (2025 - MIN(year)) estimates the rate of value loss per year.

GROUP BY year, fuel ensures depreciation is analyzed separately for each year and fuel type.

üí° Why do this? This helps understand which fuel types retain value better over time. 


-- Cars with Best Value (Price per Power) --
SELECT `Name`, selling_price, max_power, (selling_price / max_power) AS price_per_bhp
FROM `car dekho`
ORDER BY price_per_bhp ASC
LIMIT 5;

Explanation:

(selling_price / max_power) calculates the price per unit of power (bhp).

ORDER BY price_per_bhp ASC ranks cars with the best price-to-power ratio at the top.

LIMIT 5 shows only the top 5 most cost-effective cars.

üí° Why do this? This helps find cars that offer the most power for the least amount of money.
