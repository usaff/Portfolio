Data Cleaning in Power Query

Overview

This report focuses on data cleaning and transformation in Power BI/Excel using Power Query. The goal is to standardize, clean, and prepare raw data for reporting and analysis. It ensures data consistency, removes errors, and enhances overall data quality.

Key Data Sources:

  • Excel spreadsheets with raw transactional or operational data.
  • External data sources (if applicable) integrated via Power Query.

Steps Taken

1. Data Import

  • Loaded raw data into Power Query from the Excel sheet "Data Cleanning."
  • Checked data types and formats to ensure correct interpretation.

2. Data Cleaning

  • Removed duplicates to avoid redundant records.
  • Handled missing values:
    • Filled missing data where possible.
    • Removed rows with excessive missing values.
  • Standardized column names for better readability and consistency.
  • Trimmed and cleaned text fields to remove leading/trailing spaces.

3. Data Transformation

  • Split and merge columns to extract relevant information.
  • Created calculated columns for enhanced insights.
  • Formatted date/time fields to standard formats.
  • Applied data filtering to retain only relevant records.

4. Data Modeling

  • Established relationships between tables (if multiple tables are present).
  • Defined data types for optimized performance.

5. Visualization (If applicable)

  • Created summary tables and charts for data insights.
  • Applied conditional formatting for better readability.

Performance Optimizations

  • Optimized data types to reduce memory usage.
  • Removed unnecessary columns to improve query performance.
  • Reduced calculated columns in favor of measures to enhance efficiency.
  • Used query folding to push transformations back to the source when possible.

Future Improvements

  • Automate data refresh for real-time updates.
  • Implement error-handling measures to detect and log data issues.
  • Enhance data validation with additional rules and constraints.
  • Expand data sources to enrich analysis.

Best Practices & Considerations

  • Maintain a consistent naming convention for columns and queries.
  • Document all transformations for auditability and collaboration.
  • Minimize the use of calculated columns where DAX measures can be used.
  • Optimize query performance using data reduction techniques.
  • Ensure data security and compliance when handling sensitive information.
